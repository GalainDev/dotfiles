# Cheatsheet

Generated by the `cheatsheet` skill — reads the live config files fresh every run.
If a binding here looks wrong, re-run `/cheatsheet` before assuming this file is
stale; it's regenerated from source each time, not hand-edited.

## AeroSpace (window manager — `Alt`, system-wide)

| Keys | Action |
|---|---|
| `Alt+j/k/l/;` or arrows | Focus window left/down/up/right |
| `Alt+Shift+j/k/l/;` or arrows | Move window left/down/up/right |
| `Alt+,` / `Alt+.` | Focus previous/next monitor |
| `Alt+Shift+,` / `Alt+Shift+.` | Move window to that monitor and follow |
| `Alt+Ctrl+,` / `Alt+Ctrl+.` | Move whole workspace to that monitor |
| `Alt+O` / `Alt+P` | Resize smaller / larger |
| `Alt+/` | Toggle tiles layout (horizontal↔vertical) |
| `Alt+\` | Toggle accordion layout |
| `Alt+F` | Fullscreen (AeroSpace-managed) |
| `Alt+Shift+F` | Float ↔ tile toggle |
| `Alt+Shift+B` | Balance window sizes |
| `Alt+1`–`9` | Switch to workspace 1–9 |
| `Alt+Shift+1`–`9` | Move window to workspace N and follow |
| `Alt+Tab` | Jump to previous workspace |
| `Alt+Shift+R` | Reload AeroSpace config |

Workspace 9 lives on the laptop screen; 1–8 and 10 are forced to the secondary monitor.

## WezTerm (terminal — `Cmd`; works even with no multiplexer running)

Config auto-reloads on save (default behavior, not overridden). Manual reload: `Ctrl+Shift+R`.

| Keys | Action |
|---|---|
| `Cmd+H` | Split side-by-side |
| `Cmd+N` | Split top/bottom |
| `Cmd+J/K/L/;` | Focus pane left/down/up/right |
| `Cmd+W` | Close current pane (no confirm) |
| `Cmd+Shift+W` | Close current tab (no confirm) |
| `Cmd+Alt+J/K/L/;` | Resize active pane left/down/up/right |
| `Cmd+Alt+R` | Swap pane with neighbor (rotate clockwise) |

Tab commands are WezTerm defaults (untouched by config, aside from `Cmd+W` above shadowing
the stock tab-close binding — restored separately as `Cmd+Shift+W`):

| Keys | Action |
|---|---|
| `Cmd+T` | New tab |
| `Cmd+Shift+T` | New tab (default domain) |
| `Cmd+1`–`9` | Jump to tab 1–9 (9 = last tab) |
| `Cmd+Shift+]` / `Ctrl+Tab` | Next tab |
| `Cmd+Shift+[` / `Ctrl+Shift+Tab` | Previous tab |
| `Ctrl+Shift+PageUp` / `PageDown` | Move tab left/right |

Plain `Alt`/Option is untouched — types accented characters normally (é, ñ, etc.), no
composed-key override in this config.

## herdr (primary multiplexer — prefix `Ctrl+B`; run `herdr` to start)

Number row is plain US ABC layout: Shift+digit sends punctuation (`!@#...`), not a
reportable shift+digit chord, and Alt+digit hits macOS composed-character interception
*and* an ESC-timing race over plain PTY. That's why agent switching below is plain
letters, not a number-row scheme.

| Keys | Action |
|---|---|
| `C-b h` | Split side-by-side |
| `C-b n` | Split top/bottom |
| `C-b j` / `k` / `l` / `;` | Focus pane left / down / up / right |
| `C-b c` | New tab |
| `C-b &` | Close tab |
| `C-b d` | Detach (session keeps running) |
| `C-b Shift+r` | Reload herdr config |
| `C-b w` | Workspace picker (fuzzy) — **use this to jump to a space** |
| `C-b Shift+n` | New workspace (herdr default, not overridden) |
| `C-b g` | Goto (fuzzy jump) |
| `C-b y` | Copy mode (then `v`/space select, `y`/Enter copy, `q`/Esc cancel) |
| `C-b a` | Next agent |
| `C-b Shift+a` | Previous agent |
| `C-b 1`–`9` | Jump to tab 1–9 (herdr default, not overridden) |

Agent sidebar panel is grouped by workspace (not by attention priority), and agent
labels are shown directly on pane borders.

## tmux (fallback — same prefix `Ctrl+B`, only if launched directly instead of herdr)

| Keys | Action |
|---|---|
| `C-b h` | Split side-by-side (preserves cwd) |
| `C-b n` | Split top/bottom (preserves cwd) |
| `C-b j` / `k` / `l` / `;` | Focus pane left / down / up / right |
| `C-b r` | Reload tmux config |
| Mouse drag in copy mode | Copies selection to `pbcopy` on release |

Resurrect/continuum auto-save the session every 15 min, restore on tmux start.

## Neovim (leader = `Space`, mouse disabled on purpose)

| Keys | Action |
|---|---|
| `Esc` | **Saves the file** (not just exits insert mode — deliberate) |
| `Ctrl+A` | Select all |
| `<space>f` | Find files |
| `<space>s` | Search text (grep) |
| `<space>b` | Buffers |
| `<space>e` | File browser (oil.nvim) |
| `<space>g` | Git (neogit) |
| `gd` | Goto definition (Snacks picker) |
| `<space>` (then wait) | which-key popup — shows every live binding |
| `p` over a visual selection | Paste without clobbering the clipboard register |

Other notable behavior: `gitsigns` shows current-line blame inline.

## CLI Tools

**zoxide** — smart `cd` that learns your most-visited directories, wired into the
shell (`programs.zoxide` in `home.nix`).
```
z <partial-name>     # jump to best-matching visited directory
zi <partial-name>    # interactive picker if more than one match
z -                  # previous directory
```

**fzf** — fuzzy finder, shell-integrated (`programs.fzf` in `home.nix`).
```
Ctrl+R                # fuzzy-search command history
Ctrl+T                # fuzzy-insert a file path at the cursor
<cmd> | fzf            # fuzzy-filter any command's output
```

**bat** — `cat` with syntax highlighting and a git-diff gutter.
```
bat file.py            # view with highlighting + line numbers
bat -A file.txt         # show whitespace/invisible characters
bat -n file.txt          # force line numbers even when piped
```

**fd** — fast, gitignore-aware `find` replacement.
```
fd pattern              # find files/dirs matching pattern, respects .gitignore
fd -e py                # only .py files
fd -H pattern            # include hidden files too
```

**lazygit** — full git TUI (complements neogit inside nvim, `<space>g`).
```
lazygit                 # open in the current repo
  space                  # stage/unstage the file or hunk under cursor
  c                      # commit staged changes
  P / p                  # push / pull
```

**yazi** — terminal file manager with previews (complements oil.nvim, `<space>e`).
```
yazi                    # open in the current directory
  hjkl                   # navigate (vim-style)
  space                   # select a file
  y / p / d               # yank / paste / delete selected
```

**gh** — GitHub CLI.
```
gh repo create <name> --public
gh pr create
gh repo view <owner>/<repo>
```

**jq** — JSON processor.
```
cat file.json | jq '.'          # pretty-print
cat file.json | jq '.key'       # extract a field
cat file.json | jq -r '.key'    # extract as raw string, no quotes
```

**glow** — renders markdown in the terminal (styled, not raw text) — Nix-managed via `home.packages`.
```
glow file.md            # render once and print
glow                     # render README.md in the current directory
glow -p file.md          # page through with less-style scrolling
glow -s dark file.md      # force a style (auto-detects terminal bg by default)
```
